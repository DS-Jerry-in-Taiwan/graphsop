# Phase 01 - 開發目標：知識建模與數位孿生核心構建

[cite_start]**專案名稱**: TwinLex (GraphSOP) 
[cite_start]**階段**: Day 1 - 知識建模 
[cite_start]**執行模式**: 混合模式 (Mixed Mode) 

## 🎯 核心目的
[cite_start]將「智慧攝影機操作說明書」等非結構化文本，轉化為具備邏輯檢核能力的 **Property Graph (屬性圖)**，建立設備的「邏輯數位孿生」大腦 。

## 🚩 開發目標
1. [cite_start]**數據初始化**: 準備包含設備參數、SOP 流程及安全禁區的虛擬手冊文本 。
2. [cite_start]**Schema 定義**: 建立包含 `Camera`, `Action`, `Constraint`, `Standard` 等核心節點的圖譜模式 。
3. [cite_start]**自動化提取**: 利用 LLM 從文本中精確提取三元組 (S-P-O) 並寫入圖資庫 。
4. [cite_start]**實體焊接**: 執行實體對齊，確保語意模糊的術語（如「鏡頭」與「攝影機」）指向正確的數位孿生實體 。

## 📦 預期產出物
* **設備圖譜模式 (Ontology)**: 定義好的節點與關係結構。
* **初始知識圖譜**: 存放在 Neo4j 或記憶體中的設備邏輯網。
* **提取日誌**: 記錄提取過程中的實體解析與消解結果。

## ✅ 驗收標準 (Definition of Done)
* **結構完整性**: 圖譜必須包含至少 1 個物理極限限制與 1 個安全禁區限制。
* **路徑連通性**: 從「操作指令」節點必須能追蹤到對應的「驗證標準」節點。
* **實體消解率**: 核心實體（如 Camera ID）無重複節點。
